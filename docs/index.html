Nuestra propuesta

03

Nuestra propuesta:

Aprovechamos al máximo el software Front-End, creado una arquitectura que agiliza el arranque de los proyectos, asegurando buenas prácticas:

  • Creando nuestro propio software para línea de comandos (CLI)
  • Desarrollando un catálogo de componentes
  • Intregrándonos con sistemas de CI/CD (Integración y Despliegue continuo)

Catálogo de Componentes:

A lo largo de los años también hemos podido observar como a nivel visual se repiten ciertos componentes.

El departamento de Arq. Front decició crear un catálogo de componentes customizable para reducir tiempos de desarrollo y aumentar calidad.

  • Formularios: Input, select, fechas...
  • Datos: Gráficas, Mapas, Paginadores...
  • Info: Modales, diálogos, notificaciones...
  • Navegación: Desplegables, menús, árboles...

Línea de comandos:

Tras años de experiencia hemos detectado que a nivel de arquitecturas se repiten ciertos patrones.

Mediante nuestro propio software generaramos la estructura básica de un proyecto, homogeneizando desarrollos y acelerando el inicio del proyecto.

Contiene la arquitectura CORE necesaria en la mayoría de proyectos Front-End:

  • Navegación / Menú
  • Gestión de seguridad y autenticación
  • Sistema de componentes y templating
  • Herramientas de Calidad de Código

Integración con sistemas CI/CD:

CI/CD son mecanismos que permiten entregar software con una frecuencia alta, mediante el uso de la automatización en las etapas del desarrollo.

Aprovecharemos fases del desarrollo (subidas de código) para "lanzar" procesos automatizados.

  • CI (Continuous Integration): Procesos de validación (tests unitarios, lintado...)
  • CD (Continuous Delivery): Despliegues y publicación automatizados

Integración con sistemas CI/CD:

Se integran muy bien en flujos de trabajo iterativos, por ejemplo metodología SCRUM.

Generan un flujo constante de desarrollo: las pruebas de la fase de CI acaban desencadenando desarrollos que se despliegan mediante la parte de CD.

A su vez lo desplegado en CD genera nuevas tareas que arrancan de nuevo el CI.

Integración con sistemas CI/CD:

Ejemplo made in "la casa": Mates

Los desarrollos de código se integran en el proyecto mediante Pull Requests (PR)

Las PR disparan la ejecución de:

  • Pruebas de calidad de código
  • Tests unitarios
  • Despliegue en local
